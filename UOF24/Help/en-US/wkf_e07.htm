Form viewer query

Form inquirer can see all the contents of the forms he manages. He can also print, and no matter the form is closed or not, it can usually be seen.

Basic query

Select first, “form status”, which can be divided into five types: All status, in process, closed, abnormal and void. You can select according to the content to be queried.

In process: You can query the unclosed forms.
Closed: Pass means that the approval of this form has been finished and passed; reject means the approval of this form has been finished, but the application content is not passed yet.
Abnormal: When red error message appears for the form, then the approver clicks “Abnormal send”, but the form cannot be sent to the next stop, that is, abnormal status appears.
Void: Since there is trouble on the form or there is wrong filling, the form needs to be voided.

Next, select the start and end time of “application date”; you can enter “Form number start and end”, next, select “form name” to be queried, finally, click “query”, the system will then list the queried forms.

Advanced query

You can also select “advanced query”, and you need to select first “form name”, then select form revision.

Enter form field condition to be specifically queried, and you can enter at most three conditions. Click “select” to enter the setup screen, and it can be divided into normal conditions and range conditions.

Normal conditions: Not every field can be done with general conditions query. It is applicable to the fields of single line text, checking block, single choice button, date, number and pull-down menu; select first the condition for setup, then set up the OP, then select the comparison data, then click “Submit” , the system will then list the queried forms.

Range conditions:  It is applicable to fields such as date, time and quantity. Select first “condition setup”, then enter the start and end value of the comparison data, then select the OP value of < or <=, then click “Submit” , the system will then list the queried forms.

表單欄位顯示設定
根據版本的欄位設定，讓查詢後的列表可以顯示所設定的欄位；
當表單版本更新後，若與前一版本相同的欄位，系統會自動顯示，新版本的新欄位，則需要重新設定。
