Flow setup

New flow
In the form version setup, press “Flow setup”, then click “New flow”, the setup screen will then appear.
Click “insert stop” to start the setup flow, the system will then emerge window for selecting the site category, which can be divided into seven types: Approval stop, sub-flow stop, branch stop, APP stop, customized stop, selection stop and organization stop.
Approval stop: Single stop setup.

  1. You can set up if the site is to be assigned by the previous site, and this means that when the user is approving, he needs to assign the approvalmember of the next stop, and themember to be assigned can be limited.

  2. Approval type:

    • 一般：該站點只能有一名簽核者，同意即送下一站，該簽核者否決表單即結案。
    • 並簽：該站點可設定多人簽核，一人簽核同意即送下一站，會依「表單維護」的並簽設定來決定是否一人否決即可結案。
    • 會簽：該站點可設定多人簽核，全部簽核者同意才送下一站，一位簽核者否決即結案。

  3. If “Can this stop be modified” is set up as yes, it means that the user can change arbitrarily the approver of this stop.

  4. You can setup "Delay Setup", " Periodic Notify", "Skip" in for this stop, please refer to "Set up a new form" for the detail.

  5. The final step is set up the approver. Themember to be notified before approval or after approval can be set up directly as specialmember or as selectedmember.
Sub-flow stop: For the sub-flow or parallel-flow site setup, please select the flow to be used from sub-flow and parallel-flow list.
Branch stop: It is to judge the condition setup. Since it is condition setup, hence, it is needed to select first the condition to be judged, after the selection, please select sub-flow or parallel flow corresponding to judgment condition. One judgment condition can only be accompanied with one sub-flow or parallel flow.
APP stop: It provides the end for calling external application program (DLL). DLL will then decide the site flow.
Customized stop: The user is allowed to, during the movement of the form, self-make the approval flow.
Selection stop: First, the form designer set up the sub-flow, then the user select more than one sub-flows from them to be used as approval flow.
Organization stop: You can follow the selected department,member, job rank or job title as the flow judgment, press “...” to select field code, then follow the content selected from the form as the approvalmember of the site.
Notice Stop: A kind of Stop in Flow setup, it is only for notice, no any approval authorized.
Click “Submit” to add a site or flow. If you want to change, please click “setup” to change the site or flow.
Set up sub-flow starting and ending notification
In the form version setup, press “Form setup”, then click the name of the sub-flow to set up the starting and ending notificationmember of the sub-flow.
Minimum Approval Stops : Setup the minimum Approval Stops to end the Flow.
Sub-Flow Initiation Basis : Default Setting is in [BPM→Sub-Flow Design], but if you want to setup different Basis for individual form, click "By Field" and select the Organization field in the form which is selected [Department+Position] by Applicant.
Start/End Notice : setup Notified members while the Sub-Flow Start/End.
Maintain flow
In the form revision setup, click “Flow setup” , then click “Maintain flow” , the screen for the setup of main flow will then appear.
You can change the flow previously set up. You can select “Insert stop” , “Setup” and “Delete”.
Attribute setup of the site
  Each site can be used as viewing the stop, setup of web service, stop remark, stop mail, closure allowed stop, site code and approval setup.
View stop information: The same as clicking “setup”, you can change the site or flow.
Web service setup: After setup form is sent out, this is the field information that needs to be sent for the called Web Service.
Stop remark: Set up the remark description of the stop.
Stop’s mail information: You can set up the format of the notification mail in this stop.
Closure allowed stop: You can set up the case closing condition or whether this site can be closed.
Site code: It can be used as judgment within the program.
Approval setup: You can set up if the approval button is to be displayed or not (Reject/Reject/Add stop).
Note:

  1. When the form is announced and in use, the flow can still be changed to make it correct. However, if the flow has large change, we suggest adding new form version to avoid error on the executing form.

  2. When setting up the sub-flow stop, it is needed to set up sub-flow or parallel flow, please refer to sub-flow design and parallel flow design.

  3. When setting up branch stop, it is needed to set up judgment formula and condition, please refer to Condition setup.
