In process: Form is currently in process, that is, it is waiting for the approval by the approver in the next stop.

Closed: The form has been closed, which includes overruled, rejected or void form.

Abnormal: When red error message appears on the form, the approver clicks ¡§Abnormal send¡¨, but the message cannot be sent out, and abnormal status appears.

For the queried forms, different operations can be performed according to different situations. Basically, the operations can be divided into four types: Send to next stop, specify approval, unlock and void.

 

Send to next stop: When the approver click ¡§abnormal send¡¨, the administrator then clicks ¡§send to next stop¡¨ to set up the approvingmember so that the form can continue to be sent to the next stop or to be closed. And the subsequent stop will become free flow.

Specify approval: When the approver does not approve for a long time, or when the approver forgets to set up the agent, the administrator can set up for the approver so as to speed up the processing speed of the form.

void: Since there is trouble on the form, or the form is filled wrong, hence, the form needs to be rejected, and the rejected form can be deleted.
