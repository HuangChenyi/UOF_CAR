Filing query and processing

Only the person who has the right to file the form can query the related forms.

Classification query

In ¡§form list¡¨, click the form name to be queried for filing status, then select the start and end date of the application date, then click ¡§query¡¨, the system will then list the query forms.

In the query result, it will be listed with form number, form name, subject, applicant, result, application date, closure date and operate. If you click ¡§print¡¨, you can then print form and approval information. If it is set up with background picture, or if the user has set up digital signature, then during the printing, it will be printed.

You can click ¡§Mark as Processed¡¨ or ¡§Unmark¡¨ so as to change the processing results of the forms. Please refer to filing form processing in item 3.

Conditional query

Select first ¡§approval results¡¨. The status can be divided into four types: all status, pass, reject and in process. They should be selected according to the query content.

In process: You can query the unclosed forms.

Pass: This form has been signed and approved.

Reject: This form has been signed, but the application content is not passed yet.

Next, select the start and end time of ¡§application date¡¨, then enter ¡§Form number start and end¡¨, and this field can be selected not to enter; next, select ¡§form name¡¨ to be queried, and this field can remains its default value of¡§all forms¡¨, then click ¡§query¡¨, the system will then list the queried forms.

You can also click ¡§advanced selection¡¨, and you need to select ¡§form name¡¨, then select form revision.

Enter the form field conditions to be specifically queried, and at most three conditions can be set up; then click ¡§select¡¨ to enter setup screen, which can be divided into normal conditions and range conditions.

Normal conditions:  Not every field can be done with general conditions query. It is applicable to the fields of single line text, checking block, single choice button, date, number and pull-down menu; select first the condition for setup, then set up the OP, then select the comparison data, then click ¡§Submit¡¨ , the system will then list the queried forms.

Range conditions:  It is applicable to fields such as date, time and quantity. Select first ¡§condition setup¡¨, then enter the start and end value of the comparison data, then select the OP value of < or <=, then click ¡§Submit¡¨ , the system will then list the queried forms.

Filing form processing

When the filing handler sees the filing forms, he can remark on each record the processing status.

When the processing is finished, check the form and click ¡§Mark as Processed¡¨, the filing handler can then record that he has finished the processing, for example, when the human resource person checks the travel form, he can remark the travel fee behind the salary data, then click ¡§Mark as Processed¡¨, which means that the fee of the form has been checked with the salary.

Form export

Through the query interface, the form data to be exported will be queried first, then click ¡§Query¡¨, the system will then print the queried forms. For the usage of advanced selection, please refer to item 2, condition query.

The queried forms include closed and non-closed forms, and they can be exported in two Excel tables, one is statistics: all the fields will be shown in list type statistics; another is form: all the fields will be shown in single sheet statistics.

Click ¡§Export statistics Excel¡¨ or ¡§Export form Excel¡¨, the system will then follow scheduling time to transfer the document to ¡§download exported report¡¨ and send out exporting finished message to exporting user.

Download exported report

All the files exported by clicking ¡§export statistics Excel¡¨ or ¡§export form Excel¡¨ will be listed here, and they can be re-downloaded for use, if you do not need it anymore, you can check the file and delete it again.
