Method to add new and upload document(store)

The author can upload new document, can store or release the document so that the author or reader can read the document.

Text Mode support HTML editor which supporting Inserting Graphics, Attachments, Stickers and Emoji, you can minimizing the Graphics by click right button.

Tag the directory which the document to be uploaded, then go to the tool bar at the upper side and click for ¡§New doc.¡¨to enter the operation screen.

Basic attribute: Click if the document is to be stored or released, then enter the document name (you can select not to enter), then enter document number, and if you want to let the system control the revision automatically, you do not need to check ¡§Manually controlled revision¡¨, then select the issuing unit and keeper, finally, enter the summary and keyword to describe this document.

Storage time: It is effective setup and invalid setup of the document.

Confidentiality setup: You can set up the confidentiality level; if other authors of the same directory are not allowed to edit, then you can cancel the checking of ¡§Author group allowed for joint edit¡¨; if you want to convert it to  PDF file, please check ¡§Convert to PDF¡¨, then select the control status of PDF file, which can be divided into ¡§Printable¡¨,  ¡§Store as¡¨,  ¡§Copy¡¨; in the future, when the reader is downloading file, the PDF reading behavior can be controlled.

View authorization: If the administrator opens the authorization to the author, the author can then follow his own opinion to decide who can have the view request authorization and decide the view request flow; however, if it is not open, the author cannot make customized view request member.

Document category: It is to set up the attribute of this document. One document can have multiple attributes, and the document category is helpful to future search of document by category.

Reference document: When the documents to be uploaded are related to certain documents within the document center of the system, the related documents can then be seen as reference documents. Click ¡§Add in reference document¡¨ to select the related documents.

Click ¡§Browse¡¨ to select files to be uploaded, after selecting the files, please click ¡§OK¡¨ to finish.

After finishing all setups, click ¡§Submit¡¨ to upload the documents to add a new document.

Note: Store this document: After document is checked in and modified and is about to be saved back into the system, please click ¡§store¡¨.

Release this document: Release the document so that the reader can read and download, and the released revision can be changed.

Manual revision control: It means that the revision is manually entered instead of being brought out automatically.

Release unit: The release unit of hard copy document.

Keeper: The keeper of hard copy document.  There are lots of keepers, and the hard copies will be issued to these departments for keeping. For example, the technical document control center will release the documents to technical department and production department, etc. When there are new revisions, these departments will be asked to recycle for updating new revisions.

Master copy control: It is to control if the document can be re-edited, printed, saved as and copy, etc. In addition, the downloading authority of the master copy can also be controlled too. If the company is not installed with PDF Server, this control will be unable to be achieved.
