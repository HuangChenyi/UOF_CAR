Maintain Form:

Save copy: This is the function whether the user is allowed to save the copy or not. If the user is not allowed to save the copy, please check ¡§While form is being sent, copies not allowed to be saved¡¨, hence, when form is voided, it will not be allowed to be saved into the draft folder.

Batch approval: It is to set up if the approver is allowed to sign multiple forms at one time.

Modify authority level: When the form is sent to the next site, whether the form field is allowed to be changed by certain approver? If so, please select ¡§Modification allowed¡¨, or assign directly who can make the change.

Apply for orders: It is to set up whether substitute application is supported or not, that is, whether other colleague to fill in the form on behalf is allowed or not. When it is checked, it is allowed to set up whether the substitute application from colleagues or othermember in the same department is allowed or not.

Print authorization: It can be set up that form is only allowed to be printed by specificmember. If it is not checked, then allmember can print.

Print with background: It is to set up when the form is printed, whether the background picture should be printed or not. The background picture can be used as watermark effect, and if the background picture is customized, then the photo file needs to be uploaded.

Abnormal Form: When red alarm message appears on the form, is approver allowed to click ¡§Allow abnormal sent¡¨? If so, after form is abnormally sent out, administrator is only allowed to send it to the next site. All the subsequent flows will be free flows, that is, it is needed for the approver of each site to set upmember of the next site.

Form inquiry: Is it allowed that the applicant or approver can inquiremember in other non-fixed flow? This will not affect form flow, and it can still be sent to the next site after inquiry. When the person inquired has replied, the system will notify the inquirer. The range of themember to be inquired can be limited.

Form delayed: It is to set up the time-out time of this form. For example, if this form should be finished within four working days, then the time-out time is set up as four days. The system will summarize time-out form in the same message and provide form linking function.

 Skip : If the approver didn't approve the form within the setting time, the system will set as "Approved" automatically in this stop and skip the form to next stop.

Periodic Notify : System will Notify the approver periodically until the approver has approved the form. For example, if setup 2 hours, system will notify the approver every 2 hours until the form is approved.

Online application: It is to set up if online application is allowed, or if the throw-in of form content from other system is allowed. For example, for the form connected through ERP, you should check ¡§User cannot apply for forms online¡¨; When the approval of the form is returned, you can set up if the applicant is allowed to treat it in the system, or if the return is allowed.

Multi-or approval setup: It provides the setup of the case closing condition of multi-or approval rejection. That is, should the case be closed when there is rejection from single person or should the case be closed when there is rejection from allmember; when viewing approval opinion, you can set up if the information of non-approvalmember is to be displayed or not. The default value is set to display, and the approval status will display ¡§It had  approved by others¡¨.

Add stop setup: There are two options, Add-Stop & Advanced Add-Stop. It is for Applicants or Approvers add Stops by chance. Add-Stop: Applicants or Approvers can add Stops before next Stop and decide going back to self or to next Stop while added Stops are approved. Add-Stop is restricted in Normal in Approval Stop & Customized Stop. Advance Add-Stop: Applicants or Approvers can add/insert Stops in the following Stops. It is restricted in Multi-and/Multi-or/Normal in Approval Stop, Customized Stop & Notice Stop. Applicants or Approvers can only choose the restrict members if the¡iRestrict¡jis checked.

Returned setup: It provides the setup that when the application is returned, can the applicant select the site of skipping when it is sent out, and the site that is skipped will receive the notice.

Stop setting: It provides the setup whether a rejection or returning of a stop is allowed. Setup automatically skip while the approvers of following stops are the same. It will automatically approved if the following stops are in main Flow or the same sub-Flow while the approver click ¡iapprove¡j.

Setup CC: It allows the setup that when the approver is approving, whether CC or BCC is allowed or not.

Setup comments: This is the setup that when there is approval opinion during the approving process, should a new window be opened for the display.

Approval history settings: It provides the setup that should the approval history information display the job title information of the approver, and should the displayed job title include ¡§Supervisor¡¨ and ¡§Approver¡¨.

Setup Notice after closure: You can set up if the applicant is allowed to send form notice to others after closure so that others can view the content.

Supervisor Form Query setup: You can set up if the supervisor is allowed to query the forms of his subordinates.

Form Forum Setup: Active a thread to all related members for every individual form. Click Subject in [Form Thread] can view the Form.

Mobile setup: Approval is not allowed in Mobile device.

Form user authority level: It is to set up form user. Click¡§select member¡¨ to select the needed member.

Filling handler: It is to set up the form filling handler, click ¡§select member¡¨ to select the filling handler.

Form viewer: This is not form user, filling handler or approver, however, this person has the authority to check form status.

Form manager: When there is abnormity on the form, in addition to the administrator, responsible person of the form can also process the form.
